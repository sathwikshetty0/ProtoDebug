# 🔘 ProtoDebug — AI-Powered Troubleshooting System

**ProtoDebug** is a high-fidelity, interactive debugging and learning platform designed for modern developers and hardware enthusiasts. It provides a "Command Center" experience for troubleshooting complex systems across IoT, Web Applications, and Robotics.

![ProtoDebug Hero](https://images.unsplash.com/photo-1518770660439-4636190af475?q=80&w=1470&auto=format&fit=crop)

---

## 🛠️ Intelligence Domains

### 🔩 Hardware Lab (IoT)
Pinpoint hardware issues by selecting your microcontroller (ESP32, Arduino, Raspberry Pi, etc.), adding specific sensors or modules, and describing the current failure. ProtoDebug provides step-by-step circuit and voltage debugging protocols.

### 🌐 Web Application Lab
Troubleshoot modern web stacks including React, Next.js, and REST APIs. Get enterprise-grade fixes for common state management, hydration, and communication bugs.

### 🤖 Robotics Suite
Access end-to-end build guides and real-time troubleshooting for lab systems like Line Followers, Maze Solvers, and Soccer Bots.

---

## ✨ Features

- **🧠 Integrated AI Debugger**: Persistent assistant with dark mode support, minimize/maximize functionality, and interactive example queries for deep diagnostics.
- **⚡ Proactive Navigation**: Global command palette (⌘K), breadcrumb trails, and scroll-progress indicators for lightning-fast workflow.
- **🛡️ Expert Protocols**: Advanced "Expert Mode" in Hardware Lab for deep register/interface visibility and complexity-aware robotics masterclasses.
- **🔔 Live Notifications**: Integrated toast system for immediate feedback on copying, sharing, and diagnostic actions.
- **📉 History Tracking**: Automatic local persistence for "Recently Viewed" channels and progress tracking in robotics suites.
- **📱 Responsive Lab UI**: A premium adaptive design featuring custom scrollbars, glassmorphic headers, and optimized mobile layouts.

## ⚡ Quick Start

1. **Clone the repository**:
   ```bash
   git clone https://github.com/sathwikshetty0/ProtoDebug.git
   cd ProtoDebug
   ```

2. **Install dependencies**:
   ```bash
   npm install --f
   ```

3. **Deploy the Lab (Development)**:
   ```bash
   npm run dev
   ```

4. **Access the interface**: Open [http://localhost:3000](http://localhost:3000)
